# 大宜宾高考填报 AI 助手需求文档

版本：2026-06-27  
项目：`app-aidayibin`  
目标入口：`/ai/gaokao`  
产品名：大宜宾高考填报 AI 助手

## 1. 背景与目标

2026 年高考成绩已经公布，宜宾本地家长和考生进入志愿填报窗口期。大宜宾 App 当前已有 AI 写真、AI 算命等 H5 能力，用户已经能通过 App WebView 登录、生成、保存、分享。

本需求是在现有 `app-aidayibin` 中新增一个面向四川考生的高考志愿填报辅助工具，结合原生 `xuefeng-agent` 的投档数据和推荐逻辑，用 LLM 做顾问式追问、条件抽取、解释和报告生成。

首期目标不是替用户填志愿，而是帮助用户：

- 快速整理自己的分数、位次、选科、偏好和限制条件。
- 基于历年投档数据做冲稳保初筛。
- 理解每个推荐项的风险、专业匹配和取舍逻辑。
- 生成一份可以保存和分享的志愿初筛报告。
- 通过分享报告带来大宜宾 App 的裂变传播。

## 2. 已确认约束

### 2.1 范围

- 首期只服务四川考生。
- 其他省份暂不开放，即使原生数据中有其他省份，也不在第一版前端暴露。
- 用户必须登录后才能使用，登录机制参考 AI 写真和 AI 算命。
- 第一版要做顾问式聊天，不只做静态表单。
- MVP 阶段暂不做后台运营管理模块。
- 可以保存志愿报告历史。
- 需要支持分享裂变。
- LLM Key 由项目负责人提供。
- 数据和核心思路使用原生 `xuefeng-agent`，上线授权由项目负责人后续处理。

### 2.2 产品风格

面向家长和考生本人，语言要专业、可信，但可以有大宜宾文章常见的轻松感和幽默感。

风格原则：

- 不装神，不吓人，不制造焦虑。
- 可以适度使用“张雪峰式直白表达”，但不做粗暴判断。
- 给家长看的话要讲清楚风险和取舍。
- 给考生看的话要尊重个人兴趣，不只讲就业。
- 所有结论必须能回到数据和用户条件。

示例语气：

- “这个学校不是不能冲，但它属于踮脚够门槛，不能把它当稳。”
- “专业喜欢很重要，但四年学费、城市成本、未来就业，也得一起算账。”
- “位次比裸分更靠谱。分数看着热闹，位次才是报志愿时真正能上桌的硬菜。”

## 3. 服务器可行性结论

已对 `dayibin-server` 做只读检查：

- 服务器为 2 核、约 3.5GB 内存、6GB swap。
- 磁盘剩余约 36GB。
- Node.js v20.20.2、npm 10.8.2、PM2 7.0.1。
- 当前 `aidayibin-h5` 进程在线，约 46MB RSS。
- `/ai/photo` 和 `/ai/fortune` 健康检查均返回 200。
- MySQL 库 `ai_dayibin` 当前约 1.28MB，空间和表规模都很小。
- 原生 agent 数据压缩包约 29MB，解压后约 143MB，转入 MySQL 后预计可承载。

结论：

现有服务器可以支撑该 MVP。推荐把功能合并进现有 Next.js H5，不单独部署 Python 服务。

## 4. 总体方案

### 4.1 合并方式

新增独立模块：

- 页面：`/ai/gaokao`
- 分享页：`/ai/share/gaokao/{reportId}?s=...`
- API：`/ai/api/gaokao/*`
- Feature 目录：`src/features/gaokao/*`

不复用写真、算命的图片生成任务表，不修改其主链路。

### 4.2 路由隔离

高考助手与 AI 写真、AI 算命在同一个 `app-aidayibin` 项目中运行，但必须保持入口和跳转隔离：

- 不在现有 AI 首页增加高考助手卡片。
- 不在 AI 写真页面增加高考助手入口。
- 不在 AI 算命页面增加高考助手入口。
- 不从高考助手跳转到 AI 写真或 AI 算命。
- App 侧可以单独配置入口打开 `/ai/gaokao`。
- 分享报告页 CTA 只能回到 `/ai/gaokao`。

这样既复用登录、部署、MySQL、分享底座，又避免不同 AI 功能互相导流、互相影响。

### 4.3 技术原则

- 数据查询和冲稳保分档由后端规则确定。
- LLM 不直接决定推荐名单。
- LLM 只负责：
  - 多轮追问。
  - 从聊天中抽取结构化条件。
  - 解释推荐结果。
  - 生成可读报告。
  - 把专业、城市、学费、就业等取舍讲清楚。
- 所有推荐项都要展示数据来源或依据。
- 不代填志愿，不收集高考志愿系统账号密码。

## 5. 用户流程

### 5.1 入口

App AI 栏目新增入口：

名称：大宜宾高考填报 AI 助手  
地址：`https://dayibin.cn/ai/gaokao`

如果用户未登录：

- 页面展示登录态获取中。
- 调用现有 `QFH5.getUserInfo`。
- 不能获取身份时提示用户在大宜宾 App 内登录后使用。

### 5.2 首次使用流程

用户进入后看到顾问式对话页面。

第一轮由助手主动问：

- 你是物理类还是历史类？
- 今年高考分数和全省位次是多少？
- 想优先选专业、学校、城市，还是先保证录取？
- 有没有明确不想去的城市或不想读的专业？
- 家里对学费、民办、中外合作是否有要求？

用户可以自然语言输入，例如：

> 四川物理类 588 分，位次 3 万 2，想学计算机，最好成都重庆，家里不太想读民办。

系统抽取为结构化画像后展示确认卡片：

- 省份：四川
- 科类：物理类
- 分数：588
- 位次：32000
- 专业偏好：计算机、电子信息
- 城市偏好：成都、重庆
- 限制：不优先民办
- 策略倾向：未确认

用户确认后生成推荐。

### 5.3 推荐结果流程

推荐结果分为：

- 冲一冲
- 稳一稳
- 保一保

每个推荐项至少展示：

- 学校名称
- 专业或专业组
- 年份
- 历史最低分
- 历史最低位次
- 与用户位次差距
- 风险等级
- 推荐理由
- 注意事项

页面要明确提醒：

> 本结果是基于历史投档数据和 AI 分析的志愿初筛建议，不构成录取承诺。正式填报前，请以四川省教育考试院、学校招生章程和当年招生计划为准。

### 5.4 报告生成流程

用户点击“生成志愿初筛报告”后，系统保存一份报告。

报告内容：

- 考生基本画像
- 志愿策略摘要
- 冲稳保推荐清单
- 专业方向分析
- 城市和学校层级取舍
- 风险提醒
- 下一步建议

报告可再次打开、分享、删除。

### 5.5 分享裂变流程

报告完成后提供“分享”按钮。

分享目标：

- 微信好友
- 微信群
- 朋友圈，若 App 原生能力支持

分享标题示例：

- “我用大宜宾 AI 做了一份高考志愿初筛”
- “四川考生志愿怎么填？这份 AI 初筛报告可以参考”

分享描述示例：

- “输入分数和位次，自动整理冲稳保建议。结果仅供参考，正式填报请以官方信息为准。”

分享页要求：

- 不展示用户手机号、App 用户 ID、设备 ID。
- 可展示考生画像的非敏感摘要，例如“四川物理类、位次约 3.2 万”。
- 推荐清单可以展示，但如果后续需要隐私保护，可只展示摘要，引导打开 App 查看完整报告。
- 分享页底部必须有 CTA：“我也测一份志愿初筛”。
- 分享链接必须带签名参数，不能只靠 reportId 公开访问。

## 6. 功能需求

### 6.1 顾问式聊天

必须支持：

- 自然语言输入。
- 多轮追问。
- 自动识别分数、位次、科类、专业偏好、城市偏好。
- 对缺失关键信息主动追问。
- 用户可以修改条件。
- 条件确认后再生成推荐。

缺失信息处理：

- 没有科类：必须追问。
- 没有位次：可以用分数先试算，但必须提示位次更可靠。
- 没有专业偏好：先给宽泛方案，再追问“想优先就业、考研、公务员、城市还是兴趣”。
- 没有策略偏好：默认均衡，不能默认激进。

### 6.2 结构化考生画像

字段建议：

- `province`：固定四川
- `exam_year`：2026
- `subject_type`：物理类 / 历史类
- `score`
- `rank`
- `batch`
- `preferred_majors`
- `rejected_majors`
- `preferred_cities`
- `rejected_cities`
- `school_level_preference`
- `risk_preference`：激进 / 均衡 / 稳妥
- `tuition_limit`
- `accept_private`
- `accept_independent_college`
- `accept_sino_foreign`
- `accept_adjustment`
- `notes`

### 6.3 数据推荐

输入：

- 考生画像
- 原生 agent 投档数据
- 四川范围过滤

输出：

- 冲、稳、保三档推荐
- 每档数量可配置
- 结果按相关性和风险排序

分档规则第一版可以简单、可解释、可配置：

- 冲：历史位次略优于用户位次，存在机会但有波动风险。
- 稳：历史位次与用户位次接近，匹配度较高。
- 保：历史位次明显低于用户位次，主要用于兜底。

具体阈值不要写死在前端，应放后端配置中，后续根据真实使用反馈调整。

### 6.4 LLM 解释

LLM 输入必须包含：

- 用户画像
- 后端已算出的推荐清单
- 每条推荐的数据字段
- 明确约束：不得新增推荐项，不得编造分数线，不得承诺录取。

LLM 输出必须包含：

- 总体策略
- 每档解释
- 风险提醒
- 下一步建议

LLM 输出风格：

- 专业
- 清楚
- 有一点大宜宾文章式幽默
- 避免恐吓式营销
- 避免绝对化承诺

禁止表达：

- “一定能上”
- “包录取”
- “稳赚”
- “不要问学校老师”
- “官方数据不重要”

### 6.5 历史记录

用户可以查看自己的历史报告。

列表展示：

- 生成时间
- 科类
- 分数
- 位次
- 报告标题
- 推荐数量

操作：

- 查看
- 分享
- 删除

删除为软删除。

### 6.6 分享报告

分享页必须有签名校验。

分享页展示：

- 报告标题
- 生成时间
- 考生摘要
- 策略摘要
- 部分推荐结果
- 大宜宾品牌标识
- CTA：我也测一份志愿初筛

不展示：

- 用户真实 App ID
- 手机号
- 设备 ID
- 后端错误信息
- LLM 原始 prompt

## 7. 数据需求

### 7.1 数据来源

第一版使用原生 `xuefeng-agent` 的数据和推荐思路。

使用策略：

- 不直接运行原 Python 服务。
- 本地解析原生数据。
- 导入到 `app-aidayibin` 的 MySQL。
- 第一版只导入或只开放四川相关数据。

### 7.2 数据表建议

建议新增独立表，不污染写真、算命表：

- `gaokao_admission_records`
- `gaokao_consultation_sessions`
- `gaokao_reports`
- `gaokao_report_items`

`gaokao_admission_records` 字段建议：

- `id`
- `province`
- `year`
- `subject_type`
- `school_name`
- `major_name`
- `major_group`
- `batch`
- `score`
- `rank`
- `source`
- `raw_payload`
- `created_at`

关键索引：

- `(province, subject_type, rank)`
- `(province, subject_type, school_name)`
- `(province, subject_type, major_name)`
- `(year, province, subject_type)`

### 7.3 数据质量要求

- 导入后要统计四川数据总量。
- 随机抽样检查学校名、专业名、分数、位次是否正常。
- 对空位次、异常分数、重复记录做标记。
- 结果页需要显示“历史数据年份”。

## 8. LLM 配置需求

项目负责人提供 LLM 信息。

建议环境变量：

```env
GAOKAO_LLM_BASE_URL=
GAOKAO_LLM_API_KEY=
GAOKAO_LLM_MODEL=
GAOKAO_LLM_REPORT_MODEL=
```

第一版可以只配置一个模型：

```env
GAOKAO_LLM_MODEL=deepseek-v4-flash
```

如果后续要区分：

- 聊天抽取：低成本模型
- 报告生成：更强模型

则使用 `GAOKAO_LLM_REPORT_MODEL`。

## 10. 安全与合规

### 10.1 登录

- 未登录不可使用。
- 复用现有 App 登录桥和 session。
- API 必须校验 session。

### 10.2 防刷

API 要做限流：

- 聊天接口按用户限流。
- 报告生成按用户限流。
- 分享页读取不走 LLM。

### 10.3 隐私

- 不采集准考证号。
- 不采集身份证号。
- 不采集志愿系统账号密码。
- 不展示用户 App ID。
- 聊天记录和报告只给本人查看。
- 分享页只展示必要摘要。

### 10.4 免责声明

页面和报告必须展示：

> 本工具基于历史投档数据和 AI 分析生成志愿初筛建议，仅供参考，不构成录取承诺。正式填报请以四川省教育考试院、阳光高考平台和高校招生章程为准。

## 11. MVP 不做内容

第一版暂不做：

- 全国省份支持。
- 后台运营统计页面。
- 自动填报志愿。
- 志愿系统账号登录。
- 付费购买。
- 人工顾问接入。
- 小程序原生页面重写。
- 高级院校库编辑后台。

## 15. 当前建议

第一版采用“专业顾问式聊天 + 数据确定性推荐 + 可分享报告”的组合。

不要让 LLM 独立报学校，不要做全国省份，不要做后台管理。先把四川家长和考生最急需的“我这个分数和位次，到底怎么初筛”做好。

这版做稳后，再考虑：

- 全国省份。
- 后台数据统计。
- 人工顾问转接。
- 付费深度报告。
- 更多招生章程和专业就业数据。
